## Введение

**AI-PAPS** — интеллектуальная система для автоматического определения болезней и вредителей растений с использованием технологий компьютерного зрения и искусственного интеллекта.

### Назначение системы

Система предназначена для:

- **Аграриев и фермеров** — быстрая диагностика здоровья растений
- **Агрономов** — мониторинг состояния посевов на больших площадях
- **Лабораторий** — исследование и анализ заболеваний растений
- **Сельхозпредприятий** — автоматизация фитосанитарного контроля

### Преимущества

- Автоматическая диагностика по фотографиям
- Поддержка множества культур и заболеваний
- Интеграция с системами разметки (CVAT)
- Обучение собственных моделей под конкретные задачи
- Прогнозирование развития болезней

## Функциональные возможности

### 1. Распознавание болезней и вредителей

Загрузка изображения растения и получение автоматического диагноза:

- Определение типа заболевания
- Локализация поражённых областей (сегментация)
- Оценка степени поражения
- Рекомендации по лечению

### 2. Управление датасетами

Загрузка и организация изображений для обучения:

- Загрузка изображений
- Ручная и автоматическая разметка
- Импорт из CVAT
- Аугментация данных
- Разделение на train/valid/test

### 3. Обучение ИИ моделей

Создание и обучение специализированных моделей:

- Выбор архитектуры (U-Net, DeepLabV3, FPN, YOLO)
- Настройка гиперпараметров
- Мониторинг процесса обучения
- Сохранение и версионирование моделей

### 4. Каталог растений и заболеваний

Справочная система:

- База растений
- База болезней
- База вредителей
- Симптомы и признаки
- Методы лечения

### 5. Проекты

Организация работы:

- Создание проектов
- Привязка датасетов
- Управление доступом команды
- История изменений

## Руководство пользователя

### Распознавание растения

1. Перейдите в раздел **Распознавание**
2. Нажмите **Загрузить изображение** или перетащите файл
3. Дождитесь обработки изображения
4. Просмотрите результаты:
   - Тип заболевания
   - Уверенность диагноза (%)
   - Сегментированные области
5. При необходимости сохраните результат

### Создание датасета

1. Перейдите в раздел **Датасеты**
2. Нажмите **Создать датасет**
3. Заполните название и описание
4. Загрузите изображения
5. Добавьте аннотации (вручную или импортируйте из CVAT)
6. Примените аугментацию при необходимости

### Запуск обучения

1. Перейдите в раздел **Обучение**
2. Нажмите **Новое обучение**
3. Выберите модель и датасет
4. Настройте параметры:
   - Количество эпох
   - Размер батча
   - Скорость обучения
   - Оптимизатор
5. Запустите обучение
6. Следите за прогрессом в реальном времени
7. После завершения скачайте обученную модель

### Настройка CVAT

1. Перейдите в раздел **Настройки**
2. Укажите данные CVAT сервера
3. Настройте синхронизацию проектов
4. Импортируйте размеченные данные

## Типы данных

### Растения (Plant)

```
- Название (русское)
- Название (латинское)
- Вид
- Сорт (опционально)
- Описание
- Изображение
```

### Болезни (Disease)

```
- Название
- Описание
- Симптомы
- Возбудитель
- Условия развития
- Методы лечения
- Профилактика
- Изображения симптомов
```

### Вредители (Pest)

```
- Название
- Описание
- Жизненный цикл
- Признаки поражения
- Методы борьбы
- Изображения
```

### Результат распознавания (RecognitionResult)

```
- Изображение
- Дата распознавания
- Модель (версия)
- Результаты:
  - Класс (болезнь/вредитель/здоров)
  - Уверенность (%)
  - Область поражения (сегментация)
- Рекомендации
```

## Работа с системой

### best Practices

1. **Качество изображений**
   - Используйте хорошее освещение
   - Фокусируйтесь на поражённой области
   - Избегайте размытости
   - Снимайте с нескольких ракурсов

2. **Датасеты для обучения**
   - Минимум 100 изображений на класс
   - Разнообразные условия съёмки
   - Баланс классов
   - Проверка качества разметки

3. **Обучение моделей**
   - Начинайте с готовых архитектур
   - Используйте transfer learning
   - Следите за переобучением
   - Валидируйте на тестовых данных

### Ограничения

- Система предоставляет рекомендации, не заменяя эксперта
- Точность зависит от качества данных и модели
- Некоторые редкие заболевания могут быть не распознаны
- Требуется интернет-соединение для облачных моделей

### Безопасность

- Все данные хранятся в зашифрованном виде
- Доступ по ролям (admin, agronomist, viewer)
- Аудит действий пользователей
- Регулярное резервное копирование

## API интеграция

Для интеграции с внешними системами доступно REST API:

```bash
# Пример распознавания
curl -X POST https://api.example.com/v1/recognitions/ \
  -H "Authorization: Bearer <token>" \
  -F "image=@photo.jpg"

# Ответ
{
  "id": "123",
  "status": "completed",
  "results": [
    {
      "class": "late_blight",
      "name": "Фитофтороз",
      "confidence": 0.95,
      "segmentation": {...}
    }
  ]
}
```

Документация API доступна по адресу: `/v1/docs/`

## Глоссарий

| Термин | Определение |
|--------|-------------|
| **Сегментация** | Выделение областей на изображении |

| **Transfer Learning** | Использование предобученных моделей |

| **Epoch** | Один проход по всему датасету |
| **Batch** | Группа изображений для одновременной обработки |
